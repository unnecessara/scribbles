React Native vs. Ionic

A TLDR of the differences

April 25, 2019

Things I think you will agree with me on:

  1. Technology evolves fast
  2. The 'learn later' list never seems to get shorter
  3. Learning a new topic is daunting and it is difficult to know where to begin

Here's a quick tip to ease the process: Try searching for:

[the topic I want to learn] vs [a related topic I know]

Instead of reading completely new content, you get some parts that you are familiar with as well as comparisons of how the new technology in question differs or coincides. Once you have gathered a bit of information about the new technology it will push you to continue researching it or in some cases, it will let you know it is not worth your time.

For anyone coming across React Native or Ionic hoping to get a quick description of how these coincide or differ from one another, here's some information.

In a nutshell, they have a common goal to build applications for mobile devices using web technologies.

While React Native's main goal is to produce applications that look and feel as close to native applications by writing declarative code in a component-based structure, Ionic's focus lies in keeping a single codebase written with HTML/CSS/JS that runs consistently on many platforms.

It might be helpful to define the type of person that would identify more with each:

React Native

  1. You already know and love React and its component-based approach
  2. You know some iOS and Android development (useful for customizing platform-specific UI)
  3. Your main target platforms are iOS and Android
  4. You want to build large-scale applications where performance is key
  5. You want to use React Native components in an already existing application (You can do this!)
  6. You are comfortable defining layout using flexbox

Ionic

  1. You already know some Angular and or Typescript
  2. You want your app to run on many platforms (not just iOS/Android)
  3. You enjoy using Chrome dev tools
  4. You want to have one code base which will mean debugging all the target platforms at the same time
  5. You want your UI to look the same across platforms (i.e. material theme in all platforms)
  6. You care about backward compatibility (Ionic uses open web standards supported in all modern browsers)

Both frameworks have great documentation and tutorials. It's up to you to choose which adventure you want to set off on. Good Luck!
